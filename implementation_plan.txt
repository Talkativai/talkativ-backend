Frontend (React/Vite) ──── REST API ──── PostgreSQL (Neon)
    ↕                        ↕                ↕
  Stripe.js            ElevenLabs API    Prisma ORM

5. AUTHENTICATION FLOW (Secure Routes)
================================================================================

SECURITY: Auth routes use hashed paths to prevent brute-force discovery.

Env vars:
  AUTH_LOGIN_HASH    = a7f3c9d2e1b4056f8e3a9c7d2b1f4e6a8d3c5b7e9f1a2c4d6e8f0a1b3c5d7e9
  AUTH_REGISTER_HASH = b8e4d0c3f2a5167e9f4b0d8e3c2a5f7b9e4d6c8a0f2b3e5d7c9a1b4d6f8e0a2

Routes:
  POST /auth/login/a7f3c9d2e1b4056f8e3a9c7d2b1f4e6a8d3c5b7e9f1a2c4d6e8f0a1b3c5d7e9
  POST /auth/register/b8e4d0c3f2a5167e9f4b0d8e3c2a5f7b9e4d6c8a0f2b3e5d7c9a1b4d6f8e0a2
  POST /auth/refresh-token
  POST /auth/logout
  POST /auth/forgot-password
  POST /auth/reset-password/:token
  POST /auth/verify-email/:token

Registration Flow:
  1. Frontend POSTs to /auth/register/<hash> with { email, password, firstName, lastName }
  2. Backend validates with Zod, hashes password with bcryptjs
  3. Creates User record in DB
  4. Sends verification email via Resend
  5. Returns { accessToken, refreshToken, user }
  6. Frontend stores accessToken in memory, refreshToken in httpOnly cookie

Login Flow:
  1. Frontend POSTs to /auth/login/<hash> with { email, password }
  2. Backend verifies password with bcrypt.compare()
  3. If 2FA enabled → returns { require2FA: true, tempToken }
  4. If 2FA not enabled → returns { accessToken, refreshToken, user }
  5. Creates Session record (device, location, IP)

Token Strategy:
  - Access Token: 15 min expiry, stored in memory / authorization header
  - Refresh Token: 7 day expiry, stored in httpOnly cookie
  - On 401 → frontend calls /auth/refresh-token → gets new access token
  - Refresh token rotation: old token invalidated, new one issued

Password Security:
  - bcryptjs with 12 salt rounds
  - Minimum 8 characters enforced by Zod
  - Rate limited: 5 attempts per 15 min per IP

7. ONBOARDING FLOW
================================================================================

The frontend has 8 steps. Backend tracks progress via Business.onboardingStep.

Step 0 — Hear it live (demo call)
  → No backend call needed. Frontend plays a demo.
  → OR: POST /api/agent/demo-call → triggers ElevenLabs demo

Step 1 — Create account
  → POST /auth/register/<hash>
  → Creates User + empty Business shell
  → Business.onboardingStep = 1

Step 2 — Business info
  → PUT /api/onboarding/business
  → Body: { name, type, address, phone, hours }
  → Auto-fetches Google Place details if available
  → Business.onboardingStep = 2

Step 3 — Import menu
  → POST /api/menu/import/url    (if URL selected)
  → POST /api/menu/import/pdf    (if PDF uploaded)
  → POST /api/menu/import/pos    (if POS selected)
  → Parses and creates MenuCategory + MenuItem records
  → Business.onboardingStep = 3

Step 4 — AI voice & script
  → PUT /api/agent/voice   → { voiceId, voiceName }
  → PUT /api/agent/script  → { openingGreeting, closingMessage }
  → Creates ElevenLabs agent via API
  → Business.onboardingStep = 4

Step 5 — Phone number
  → POST /api/agent/provision-phone
  → Provisions number via ElevenLabs phone API
  → Assigns to agent
  → Business.onboardingStep = 5

Step 6 — Test call
  → POST /api/agent/test-call
  → ElevenLabs initiates outbound test call
  → Frontend shows live status
  → Business.onboardingStep = 6

Step 7 — Choose plan
  → POST /api/billing/subscribe
  → Body: { plan: "STARTER" | "GROWTH", paymentMethodId }
  → Creates Stripe customer + subscription (14-day trial)
  → Business.onboardingStep = 7
  → Business.onboardingDone = true

--- 8.1 Agent Lifecycle ---

CREATE agent (during onboarding Step 4):
  POST https://api.elevenlabs.io/v1/convai/agents/create
  Headers: { xi-api-key: ELEVENLABS_API_KEY }
  Body:
    {
      name: "Aria — Tony's Pizzeria",
      conversation_config: {
        agent: {
          prompt: { prompt: <system_prompt> },
          first_message: <opening_greeting>,
          language: "en"
        },
        tts: { voice_id: <selected_voice_id> }
      },
      platform_settings: {
        widget: { variant: "compact" }
      }
    }

UPDATE agent (when user changes voice/script/rules):
  PATCH https://api.elevenlabs.io/v1/convai/agents/{agent_id}

DELETE agent (when business is deleted):
  DELETE https://api.elevenlabs.io/v1/convai/agents/{agent_id}

--- 8.2 Tool Endpoints (called BY ElevenLabs during calls) ---

These are webhook URLs that ElevenLabs calls mid-conversation.
They MUST respond under 1.5 seconds.

POST /api/public/catalogue-lookup
  → Receives: { query, business_id }
  → Searches MenuItem table, returns matching items
  → Response: { items: [{ name, description, price }] }

================================================================================
13. SECURITY ARCHITECTURE
================================================================================

Layer 1 — Route Obfuscation:
  - Auth routes use long hash paths (/auth/login/<64-char-hash>)
  - Hashes stored in .env, not hardcoded
  - Any request to /auth/login without valid hash → 404

Layer 2 — Rate Limiting:
  - Auth routes: 5 requests / 15 min per IP
  - API routes: 100 requests / 15 min per user
  - Webhook routes: 50 requests / min

Layer 3 — Input Validation:
  - All request bodies validated with Zod schemas
  - SQL injection prevented by Prisma (parameterized queries)
  - XSS prevented by not rendering user HTML

Layer 4 — Authentication:
  - JWT access tokens (15 min, in memory)
  - Refresh tokens (7 day, httpOnly cookie, rotated on use)
  - bcryptjs password hashing (12 rounds)
  - Optional 2FA via TOTP (authenticator app)

Layer 5 — Security Headers:
  - Helmet.js (HSTS, noSniff, CSP, etc.)
  - CORS limited to FRONTEND_URL

Layer 6 — Session Management:
  - Track active sessions (device, IP, location)
  - Allow users to revoke individual sessions
  - Refresh token invalidation on password change
